Question-2: What is the difference between innerHTML, innerText, and textContent?

innerHTML

  • Retrieves or sets the HTML content of an element.
  • It parses and renders any HTML tags inside the string.
  • If used for setting content, it can execute scripts inside script tags.

innerText

  • Retrieves or sets only the visible text of an element.
  • It ignores any hidden elements.
  • It does not parse HTML tags. It treats them as plain text.

Question-3: What is event delegation in the DOM?

Event delegation in JavaScript is a pattern that efficiently handles events.

Events can be added to a parent element instead of adding to every single element.

It refers to the process of using event propagation (bubbling) to handle events at a higher level in the DOM than the element on which the event originated.

Question-4: What is event bubbling in the DOM?

In the DOM (Document Object Model), "event bubbling" refers to the behavior where when an event occurs on an element, it triggers not only on that element but also propagates upwards through the DOM hierarchy, reaching its parent elements and ancestors until it reaches the topmost element, allowing event handlers attached to parent elements to also react to the event that originated on a child element.
